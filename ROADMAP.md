# Roadmap do Pragmatismo Cívico

## Visão geral

Este roadmap organiza a evolução do Pragmatismo Cívico de uma base conceitual para um framework aberto, testável e aplicável em contextos reais.

## Fase 1 — Fundação conceitual

**Objetivo:** consolidar identidade, princípios e método.

### Entregas

- [x] README do projeto;
- [x] especificação inicial;
- [x] manifesto;
- [x] carta de princípios;
- [x] modelo de governança;
- [x] estrutura de indicadores;
- [x] roadmap inicial;
- [ ] glossário oficial;
- [ ] código de conduta;
- [ ] guia de contribuição;
- [ ] licença adequada ao conteúdo e às ferramentas.

### Critério de conclusão

Os documentos centrais devem ser coerentes entre si, compreensíveis ao público e suficientemente precisos para orientar contribuições.

## Fase 2 — Metodologia aplicada

**Objetivo:** transformar princípios em instrumentos de uso prático.

### Entregas previstas

- matriz de avaliação de políticas públicas;
- ficha padrão de problema público;
- ficha de alternativas;
- modelo de teoria da mudança;
- ficha de indicadores;
- protocolo de revisão periódica;
- checklist de transparência;
- modelo de relatório de decisão.

### Critério de conclusão

Uma equipe externa deve conseguir utilizar os modelos para analisar uma política sem depender de orientação informal dos autores.

## Fase 3 — Casos-piloto

**Objetivo:** testar o framework em problemas concretos.

### Domínios iniciais sugeridos

- mobilidade urbana;
- saúde básica;
- educação;
- saneamento;
- transformação digital de serviços;
- resiliência climática;
- segurança viária.

### Entregas previstas

- seleção transparente de casos;
- aplicação completa do ciclo de governança;
- publicação de dados, hipóteses e limitações;
- revisão independente;
- relatório de lições aprendidas;
- atualização do framework com base nos testes.

### Critério de conclusão

Ao menos três casos, em domínios distintos, devem produzir documentação auditável e melhorias identificáveis no método.

## Fase 4 — Comunidade e governança aberta

**Objetivo:** permitir evolução plural, organizada e íntegra.

### Entregas previstas

- processo formal de propostas de mudança;
- governança de versões;
- política de revisão e aprovação;
- conselho consultivo plural;
- regras sobre conflitos de interesse;
- fóruns de discussão pública;
- programa de colaboradores e revisores.

### Critério de conclusão

O projeto deve conseguir receber, avaliar e incorporar contribuições sem perder coerência metodológica ou ser capturado por interesses partidários.

## Fase 5 — Plataforma pública

**Objetivo:** tornar o conhecimento acessível e reutilizável.

### Entregas previstas

- portal no GitHub Pages;
- biblioteca de documentos e ferramentas;
- painel de indicadores demonstrativo;
- catálogo de estudos de caso;
- versões para impressão;
- materiais educativos;
- tradução para outros idiomas.

### Critério de conclusão

Cidadãos, gestores, pesquisadores e organizações devem conseguir localizar, compreender, baixar e aplicar os materiais com facilidade.

## Fase 6 — Rede de aplicação

**Objetivo:** apoiar experimentação em instituições e territórios.

### Entregas previstas

- parcerias com universidades e centros de pesquisa;
- colaboração com organizações da sociedade civil;
- projetos com administrações públicas interessadas;
- formação de facilitadores;
- publicação de avaliações independentes;
- comunidade internacional de práticas.

### Critério de conclusão

O framework deve possuir aplicações documentadas, avaliações externas e uma rede capaz de sustentá-lo para além de seus fundadores.

## Prioridades imediatas

1. criar `GLOSSARIO.md`;
2. criar `CONTRIBUTING.md`;
3. criar `CODE_OF_CONDUCT.md`;
4. definir licença;
5. desenvolver a matriz de avaliação de políticas;
6. criar o primeiro estudo de caso demonstrativo;
7. preparar a estrutura do portal.

## Princípios de execução do roadmap

- avançar em etapas verificáveis;
- documentar decisões e mudanças;
- testar antes de escalar;
- declarar limitações;
- preservar abertura à revisão;
- evitar crescimento institucional sem capacidade real;
- medir o valor público produzido pelo próprio projeto.

## Versionamento sugerido

- `0.1` — fundação conceitual;
- `0.2` — ferramentas metodológicas iniciais;
- `0.3` — primeiros casos-piloto;
- `0.4` — governança comunitária;
- `0.5` — portal e biblioteca pública;
- `1.0` — framework estável, testado e documentado.

Este roadmap é vivo. Sua evolução deve refletir evidências, aprendizado prático e contribuições públicas fundamentadas.
